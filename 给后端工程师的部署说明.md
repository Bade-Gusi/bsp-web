# 背水对战平台 — 后端部署工程师手册

## 一、项目总览

这是一个完整的 CS2 对战平台，包含三个端：

| 端 | 技术栈 | 代码位置 |
|------|--------|---------|
| Web 前端 | Next.js 14 + React 18 + Tailwind CSS | `bsp-web/` |
| Web API 后端 | ASP.NET Core 9 + EF Core + MySQL | `BeiShui.Server/src/BeiShui.ApiGateway/` |
| 桌面客户端 | WPF .NET 9 | `wpf/` |

**仓库地址：** `https://github.com/Bade-Gusi/bsp-web`
**分支：** `main`

---

## 二、后端部署步骤

### 2.1 环境要求

- .NET 9.0 SDK（https://dotnet.microsoft.com/download）
- MySQL 8.0+
- Redis（可选，仅匹配队列需要）
- Node.js 20+（仅前端需要）

### 2.2 拉取代码

```bash
git clone https://github.com/Bade-Gusi/bsp-web.git
cd bsp-web
```

### 2.3 配置数据库

MySQL 中创建数据库：

```sql
CREATE DATABASE beishui CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2.4 修改配置

编辑 `BeiShui.Server/src/BeiShui.ApiGateway/appsettings.json`：

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "server=你的IP;port=3306;database=beishui;user=你的用户名;password=你的密码"
  },
  "Jwt": {
    "Key": "生成一个至少32位的随机字符串，不要用默认值",
    "Issuer": "BeiShui",
    "Audience": "BeiShuiClient"
  },
  "AntiCheat": {
    "Secret": "生成一个随机密钥"
  },
  "Redis": {
    "Connection": "127.0.0.1:6379"
  },
  "Urls": "http://0.0.0.0:5000"
}
```

### 2.5 初始化数据库

```bash
cd BeiShui.Server/src/BeiShui.ApiGateway

# 安装 EF Core 工具（如果没有）
dotnet tool install --global dotnet-ef

# 创建数据库迁移
dotnet ef migrations add InitialCreate

# 应用迁移到数据库
dotnet ef database update
```

### 2.6 启动后端

```bash
dotnet run
```

默认监听 `http://0.0.0.0:5000`，启动后 API 即可访问。

可以通过 `curl http://localhost:5000/health` 验证是否启动成功，返回 200 即正常。

### 2.7 启动前端（可选，仅调试需要）

新开一个终端：

```bash
cd bsp-web
npm install
npm run dev
```

前端默认 `http://localhost:3000`，会自动通过 next.config.js 中的 rewrite 代理将 `/api/*` 请求转发到后端 `http://localhost:5001`。

> 注意：rewrite 配置中代理端口是 5001，如果后端跑在 5000，需要修改 `next.config.js` 中的 `destination` 端口。

## 六、常见问题

### Q: 启动报错 "No migrations were applied"
先确保 MySQL 连接地址正确，然后执行：
```bash
dotnet ef database update
```

### Q: 前端页面空白 / API 请求 404
检查 `next.config.js` 中的 API 代理 `destination` 端口是否和后端实际端口一致（默认 5001 vs 5000）。

### Q: CORS 跨域报错
后端已经在 Program.cs 中配置了 `AllowAnyOrigin()`，默认允许所有来源。如部署到生产环境可以限制具体域名。

### Q: JWT Token 过期
Token 有效期 7 天，如需调整修改 AuthController.cs 中的 `expires: DateTime.UtcNow.AddDays(7)`。

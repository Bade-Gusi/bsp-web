# 背水对战平台 Web 端 — 后端对接指南

## 项目信息

- **仓库**: `https://github.com/Bade-Gusi/bsp-web`
- **分支**: `main`
- **Web目录**: `bsp-web/`
- **WPF目录**: `wpf/` (同仓库内)

## 启动方式

```bash
cd bsp-web
npm install
npm run dev      # 开发模式 http://localhost:3000
npm run build    # 生产构建
npm start        # 生产模式 http://localhost:3000
```

## 环境变量

根目录创建 `.env.local`：

```
NEXT_PUBLIC_API_URL=http://你的后端地址:5000
```

不配置则默认走 `http://localhost:5001`。

## 必须实现的 API 接口

### 1. 用户认证

| 方法 | 路径 | 说明 | 请求体 | 响应 |
|------|------|------|--------|------|
| POST | `/api/auth/login` | 登录 | `{ username, password, machineFingerprint?, hardwareInfo? }` | `{ token, user }` |
| POST | `/api/auth/register` | 注册 | `{ username, password, nickname, phone?, steamId? }` | `{ message }` |
| POST | `/api/auth/steam/login` | Steam登录 | `{ steamId }` | `{ token, user }` 或 `{ needRegister, steamId }` |
| GET | `/api/auth/profile` | 获取用户信息 | — | `{ id, username, nickname, mmr, ... }` |
| PUT | `/api/auth/profile` | 更新资料 | `{ nickname?, avatarUrl?, phone?, birthday? }` | `{ message }` |
| POST | `/api/auth/forgot-password` | 发送重置验证码 | `{ username, email? }` | `{ message }` |
| POST | `/api/auth/verify-code` | 验证重置码 | `{ username, code }` | `{ message }` |
| POST | `/api/auth/reset-password` | 重置密码 | `{ username, code, newPassword }` | `{ message }` |

## SignalR Hub 对接

| Hub | 路径 | 客户端事件 | 说明 |
|-----|------|-----------|------|
| MatchHub | `/hubs/match` | `OnQueueStatus`, `OnMatchFound`, `OnServerReady`, `OnMatchAccepted`, `OnMatchRejected` | 匹配流程 |
| ChatHub | `/hubs/chat` | `OnPrivateMessage`, `OnRoomMessage`, `OnFriendStatusChanged` | 聊天 |
| CallHub | `/callhub` | `UserJoined`, `UserLeft`, `ReceiveSignal`, `ScreenShareStarted`, `ScreenShareStopped` | 语音/屏幕共享 |
| BroadcastHub | `/hubs/broadcast` | `OnServerBroadcast` → `{ serverAddress, adminName, timestamp }` | 管理员广播 |

## API 返回格式要求

所有接口统一返回格式：

```json
// 成功
{ "token": "...", "user": { ... } }

// 失败
{ "error": "错误信息" }
```

状态码：200 成功，400 参数错误，401 未授权，404 不存在，429 频率限制。

## JWT Token

- 算法: HMAC-SHA256
- Header: `Authorization: Bearer {token}`
- 有效期: 7天
- Claims: `nameidentifier` (userId), `name` (username)

## Node.js 后端快速启动示例

```javascript
// server.js
const express = require('express')
const app = express()
app.use(express.json())

// 所有路由实现...

app.listen(5001, () => console.log('API running on :5001'))
```

前端通过 Next.js Rewrite 代理转发到后端（端口 5001），已配置在 `next.config.js`。

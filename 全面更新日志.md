### 1.2 客户端反作弊 — VAC 安全重写

**涉及文件：** `AntiCheat.cs`, `AntiCheatMonitor.cs`

#### 背景
原 `CheckCS2ProcessProtection()` 调用 `OpenProcess(cs2.exe, PROCESS_VM_READ)` 枚举 CS2 进程模块，这是 VAC 检测外部工具的红线操作。

#### 删除的高风险检测
| 模块 | 原功能 | 删除原因 |
|------|--------|---------|
| `CheckCS2ProcessProtection` | OpenProcess 打开 cs2 读模块 | **触发 VAC** |
| `CheckSelfDefense` | OpenProcess 扫描自身 DLL | OpenProcess 风险 |
| `CheckTimingAttack` | 时序攻击检测 | 误报高，价值低 |
| `CheckVTd` | CPU 虚拟化检查 | 与反作弊无关 |
| `CheckMemoryIntegrity` | 读 CS2 线程/内存 | OpenProcess 风险 |

#### 保留的安全检测
- ✅ 进程名枚举 `Process.GetProcesses()` — 只读名，不 OpenProcess
- ✅ 窗口检测 `EnumWindows()` — 系统 API
- ✅ 程序集反射完整性检查
- ✅ 调试器检测 `IsDebuggerPresent()`

#### 监控去弹窗化
- `AntiCheatMonitor` 从每 30-90 秒弹验证窗口 → 每 120 秒后台静默检查，只写日志

## 二、Web API 服务 (BeiShui.AntiCheat)

### 2.1 反作弊 API 重写

**涉及文件：** `AntiCheatController.cs`, `AntiCheatStore.cs`（新增）, `Program.cs`

| 改进项 | 原来 | 现在 |
|--------|------|------|
| 违规存储 | `Console.WriteLine` 完事 | **JSON 文件持久化** |
| 限流 | 无 | 内存限流 30次/分钟/IP |
| 查询接口 | 无 | `GET /api/anticheat/violations` |
| 健康检查 | 无 | `GET /api/anticheat/health` |
| 数据库依赖 | 有 EF Core 模型 | **零数据库，纯 JSON** |

#### 新增接口
- `POST /api/anticheat/heartbeat` — 心跳上报（限流+签名验证）
- `POST /api/anticheat/report` — 违规报告（JSON 文件存储）
- `GET /api/anticheat/violations` — 查询最近违规
- `GET /api/anticheat/health` — 健康检查

## 三、CS2 服务端 BAC 反作弊插件

### 3.1 宽松化配置 + 渐进执法

**涉及文件：** `bac.cs`

#### 默认参数变更
| 参数 | 旧值（严格） | 新值（宽松） |
|------|------------|------------|
| HeartbeatTimeout | 30 秒 | **60 秒** |
| MaxMissedHeartbeats | 3 | **5** |
| GracePeriod | 15 秒 | **30 秒** |
| MaxWarnings | 3 → 永久封禁 | **5 → 临时封禁, 10 → 永久** |
| TempBanMinutes | 120 分钟 | **60 分钟** |
| ChallengeInterval | 5 心跳/次 | **10 心跳/次** |

#### 执法逻辑重写
| 阶段 | 旧逻辑 | 新逻辑 |
|------|--------|--------|
| 第 1~3 次超时 | 立即踢出 | **仅聊天警告** |
| 第 4~5 次超时 | — | 记录违规 + 警告 |
| 第 6+ 次超时 | 踢出 | 踢出 |
| 累计 5 次违规 | 永久封禁 | **临时封禁 60 分钟** |
| 累计 10 次违规 | — | 永久封禁 |

#### 新增管理员命令
- `css_bac_reload` — 重载配置
- `css_bac_status` — 查看状态
- `css_bac_unban <SteamID64>` — 解封
